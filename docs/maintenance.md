# Техническое сопровождение и восстановление

## Запуск

```powershell
npm install
npm start
```

Если порт `3000` занят:

```powershell
$env:PORT=3001
npm start
```

## Проверка работоспособности

```text
http://localhost:3000/api/health
```

Ожидаемый результат: JSON со статусом `ok`, временем работы и именем файла данных.

## Резервное копирование

Администратор может получить резервную выгрузку:

```text
GET /api/backup
Authorization: Bearer <token>
```

Выгрузка не содержит хеши и соли паролей.

## Восстановление данных

1. Остановить сервер.
2. Сохранить текущий файл `data/data.json` в отдельную копию.
3. Заменить `data/data.json` корректной резервной копией.
4. Запустить сервер повторно.
5. Проверить `/api/health` и вход администратора.

## Тестирование после восстановления

```powershell
npm test
```
