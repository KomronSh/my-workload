# Техническое задание

## 1. Наименование

Веб-приложение для оценки загруженности подразделений предприятия и перераспределения задач.

## 2. Заказчик

ИП Брюхачев Александр Александрович.

## 3. Цель разработки

Создать информационную систему, которая позволяет учитывать подразделения, задачи и трудоемкость работ, оценивать текущую загрузку подразделений и перераспределять задачи между подразделениями.

## 4. Пользователи

- Администратор: управляет подразделениями, задачами, перераспределением, аналитикой, аудитом и резервной выгрузкой.
- Пользователь: просматривает сводные данные после регистрации и входа.

## 5. Функциональные требования

- Регистрация, вход, выход и восстановление пользовательской сессии.
- Просмотр подразделений, вместимости, назначенных часов и процента загрузки.
- Просмотр задач, их статуса, трудоемкости и текущего подразделения.
- Добавление подразделений и задач администратором.
- Ручное перераспределение задачи между подразделениями.
- Автоматическое перераспределение задач из перегруженных подразделений в свободные.
- Журналирование действий с задачами и подразделениями.
- Сбор статистики работы приложения: просмотры страниц, API-запросы, ошибки, среднее время ответа по маршрутам.
- Health-check для технического сопровождения.
- Резервная выгрузка данных без хешей паролей.
- API-аудит безопасности для администратора.

## 6. Нефункциональные требования

- Серверная часть: Node.js, Express, CORS.
- Клиентская часть: HTML, CSS, JavaScript без сборки.
- Хранение данных: JSON-файл в каталоге `data`.
- Запуск: `npm start`.
- Тестирование: `npm test`.
- Порт задается переменной окружения `PORT`.

## 7. Требования к интерфейсу

Интерфейс должен содержать страницы входа/регистрации и рабочую панель с разделами статистики, подразделений, задач, перераспределения, управления, журнала активности и аналитики приложения.

## 8. Требования к SEO и продвижению

- Заполнить meta description и keywords для основной страницы.
- Закрыть страницу входа от индексации.
- Добавить `robots.txt` и `sitemap.xml`.
- Использовать понятный заголовок страницы и русскоязычное описание назначения приложения.
- Подготовить базовый план продвижения через публикацию, демонстрационное описание и мониторинг статистики.

## 9. Требования к безопасности

- Ограничить административные операции авторизацией и ролью администратора.
- Проверять входные данные на сервере.
- Использовать защитные HTTP-заголовки.
- Хешировать новые пароли с солью.
- Не отдавать хеши паролей в резервной выгрузке.

## 10. Критерии приемки

- Приложение запускается командой `npm start`.
- Пользователь может зарегистрироваться и войти.
- Администратор может создать задачу и подразделение.
- Администратор может выполнить ручное и автоматическое перераспределение.
- В интерфейсе отображается статистика работы приложения.
- Тесты проходят командой `npm test`.
- В проекте присутствуют документы по ТЗ, сопровождению, безопасности, размещению и продвижению.
